## 2. 痛点分析

### 用户痛点
1. **一份简历海投，回复率低** — 无法针对不同岗位定制内容，错失面试机会
2. **手动修改简历耗时耗力** — 每次改 JD 都要重新调整简历，效率极低
3. **不知道岗位真正想要什么** — JD 读不懂，抓不住重点，匹配无方向
4. **简历版本管理混乱** — 多个版本文件名混乱，不知道哪个对应哪个岗位

### 解决方案
| 痛点 | 解决方案 |
|-----|---------|
| ❌ 一份简历海投 | ✅ JD 分析后针对性改写，一岗一版本 |
| ❌ 手动修改耗时 | ✅ AI 自动分析，拖拽卡片秒级完成 |
| ❌ JD 读不懂 | ✅ AI 提炼核心要求，优先级分级可视化 |
| ❌ 版本管理混乱 | ✅ 树形版本管理，母版→岗位版本 |

### 价值主张
> 💡 让每一份简历都为目标岗位量身定制，求职效率提升 **3-5x**

## 3. 核心功能

### 三大功能模块

#### 📄 简历解析
- 支持格式：PDF / Word / 图片

## 5. 技术栈

### 前端技术
| 类别 | 技术 |
|-----|------|
| 核心框架 | HTML5 + TailwindCSS、JavaScript ES6+ (无框架) |
| 第三方库 | SortableJS · Tesseract.js · PDF.js |

| 类别 | 技术 |
|-----|------|

| 方案 | 说明 |

## 6. API 接口

| 接口 | 方法 | 功能 | 输入 | 输出 |
|-----|------|------|------|------|
| `/api/parse` | POST | 简历解析 | file | basicInfo, education, experiences |

| `/api/chat` | POST | AI 对话 | message, history[] | reply |

## 7. 数据结构

### ResumeDocument
```typescript
interface ResumeDocument {
  name: string
  contact: string
  sections: Section[]  // type: education | experience | skills | summary
}
```

### ResumeBase
```typescript
interface ResumeBase {
  id: string
  title: string
  pinned: boolean
  updatedAt: string (ISO)
  latestContentJson: ResumeDocument
  collapsed: boolean
  searchText: string
}
```

### ResumeVersion
```typescript
interface ResumeVersion {
  id: string
  baseResumeId: string  // 关联母版
  title: string
  targetCompany: string
  targetRole: string
  pinned: boolean
  contentJson: ResumeDocument
  jdSnapshot: JDAnalysis  // JD 快照
  searchText: string
}
```

### JDAnalysis
```typescript
interface JDAnalysis {
  companyPosition: string  // 公司 | 岗位
  summary: string  // 公司简介
  tagTitle: string  // 核心要求标签
  tagDesc: string
  coreDuties: string[]
  mustHave: string[]  // 高优先级
  niceToHave: string[]  // 低优先级
  domainKeywords: string[]
  hiddenSkills: string[]
  keyRequirements: Requirement[]
}
```

### 数据关系
> 📌 一对多关系：一个母版简历可派生多个岗位版本
> 🔗 版本继承：岗位版本继承母版内容，可独立修改
> 💾 JD 快照：每个版本保存独立的 JD 分析结果

## 9. 用户交互流程

### 7 步流程

```
上传简历 → AI 解析 → 输入 JD → JD 分析 → 拖拽改写 → 创建版本 → 导出 PDF
   ↓          ↓          ↓          ↓          ↓          ↓          ↓
 PDF/Word   结构化     粘贴/上传   提取核心    2×3 矩阵    树形管理    高清 A4
 图片       提取       岗位描述    要求        6 种变体    岗位版本    PDF
```

### 详细步骤

| 步骤 | 操作 | 说明 |
|-----|------|------|
| 1 | 上传简历 | 支持 PDF、Word、图片格式，前端 OCR 识别 |
| 2 | AI 解析简历 | 调用大模型提取基本信息、教育背景、项目经历、技能 |
| 3 | 输入/粘贴 JD | 直接粘贴岗位描述文本，或上传 JD 文件自动解析 |
| 4 | JD 智能分析 | AI 提取 3-5 个核心要求，识别公司/岗位信息，优先级分级 |
| 5 | 拖拽卡片改写 | 拖拽 JD 卡片到简历，AI 生成 6 种变体，选择最优版本 |
| 6 | 创建岗位版本 | 从母版简历派生岗位版本，保存 JD 快照，树形结构管理 |
| 7 | 导出 PDF 投递 | 样式编辑模式调整排版，导出高清 PDF，开始投递 |
